Background

近年では、カメラを使用した見守りシステムが多く存在します。なぜカメラを使用したものが増えているかというと、視覚的情報として正確に情報を伝えることができるからです。
例として、カメラを使用した見守りシステムを3つ紹介します。

ペットを対象とした見守りシステム


Qwatch I-O DATA：Link

盗難防止用の見守りシステム


VL-WD712K Panasonic：Link

人を対象とした見守りシステム


KX-HJC200K-W Panasonic：Link

今回はこれらの中から 人を対象とした見守りシステム について焦点を当てて考えていきました。人を対象とした見守りシステムには、見守る人 と 見守られる人 の2つの立場があります。

これらの人が見守りシステムを使用する理由
見守る人

  • 必要な情報を得るために使用。

見守られる人

  • 必要な情報を相手に伝えるために使用。

これらのことから考えられることは、人を対象とした見守りに カメラが本当に必要あるのか ということです。

カメラを使用する必要がない理由として

  • 情報を事細かに伝えることができるため、見守られる側の人のプライバシーがないこと。
  • 見守る側の人は、見守りたい時間ずっと画面の前にいて監視していなければならないこと。

これらが挙げられます。

Theme

そこで今回私が研究するテーマは、

視覚的情報ではなく、文章化した情報で伝達する。

ということです。
映像などの視覚的情報では、事細かに相手に伝えてしまうため、違った方法を用いて文章化し、それを伝達することで問題の解決ができると考えました。

System

  1. センサー情報を取得

  2. 文章化する

  3. Twitterへ投稿

  4. 相手に情報を伝える

Prototype

使用したセンサー

  • 温湿度気圧センサー
  • 照度センサー
  • 加速度センサー
  • 音センサー
  • 赤外線センサー

Twitterへの投稿

Twitterへの投稿は、大きく分けて二つの手順に分けて行いました。

1つ目は、インターネットへの接続です。これは、無線付きモジュールである ESP-WROOM-02 を使用しました。IDEは、Arduinoを使用しインターネットへ接続しました。

2つ目は、Twitterへの投稿です。Twitterへの接続は、Twitterが公開しているAPIを使用して行いました。

テストの様子

今回実装できた内容

  • 電気の消灯
  • エアコンのオンオフ
  • 窓の開閉
  • ドアの開閉

Task

  • 環境が変わっても柔軟に動作できるようにしなければならない。

Prospect
